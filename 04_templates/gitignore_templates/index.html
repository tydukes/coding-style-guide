    Best Practices¶

    General Rules¶

    1. Version Control Sensitive Data: Never commit:
    2. API keys, passwords, tokens
    3. Private keys, certificates
    4. Database credentials

    5. Environment variables with secrets

    6. Exclude Build Artifacts:

    7. Compiled code
    8. Distribution packages

    9. Dependency directories

    10. Ignore IDE Files:

    11. Personal workspace settings

    12. Project-specific IDE configurations

    13. Keep It Updated:

    14. Review and update .gitignore regularly
    15. Add new patterns as project evolves

    Using .gitignore¶

    # Check if a file would be ignored
git check-ignore -v path/to/file

# Remove already tracked files from git
git rm --cached <file>
git rm -r --cached <directory>

# Force add an ignored file (use carefully)
git add -f <file>

    Global .gitignore¶

    # Set global .gitignore for all repositories
git config --global core.excludesfile ~/.gitignore_global

# Create global .gitignore
cat > ~/.gitignore_global <<EOF
.DS_Store
.vscode/
.idea/
*.swp
EOF
